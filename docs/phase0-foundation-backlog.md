# Phase 0 — приоритетный техдолг (сводка)

Полные чекбоксы — в [roadmap-checklist.md](roadmap-checklist.md) **Phase 0**. Здесь — сжатый список **незакрытых** направлений для параллели с эпиком Web3.

## 0.1 Инфраструктура

- **Custom Controller (CRD `Cell`, `mmo.io/v1`):** Kubebuilder/controller-runtime — создание/удаление Pod, события в NATS — см. раздел *«Custom Controller для управления сотами»* в роадмапе.
- **NATS JetStream:** кластерный брокер (вне репо по вашему пайплайну); **subscriber** с reconnect; критерий «два сервиса обмениваются через NATS» — см. *Message Bus* в роадмапе.
- **Helm / доп. компоненты:** по политике команды (пункт «Доп. компоненты» в роадмапе §0.1).

## 0.1–0.2 Данные и симуляция

- **ScyllaDB:** клиент из Go — инфра уже в кластере; использование приложения — позже (роадмап *«Базы данных»*).
- **Репликация:** `NetworkReplicationSystem` и приоритизация — закрыты; adaptive frequency закрыт как MVP (server-only интервалы AOI, см. `backend/docs/adaptive-replication-mvp.md`). Дальше — тюнинг алгоритма под реальную нагрузку.
- **Клиент-отладчик** (Unity): базовый рендер, INetworkManager, интерполяция — роадмап §0.2 *(playground уже закрывает часть сценариев; чеклист исторический — сверять с фактом)*.

## 0.4 Grid и первая сота

- **Первая сота:** исполняемый сплит / полная live-миграция сущностей — роадмап *«Первая сота»*.
- **Grid Manager:** политики сплита/orchestration из grid-manager — роадмап *«Grid Manager»*; мониторинг нагрузки и жёсткие пороги — закрыто в роадмапе (*cell probe + `MMO_GRID_THRESHOLD_*`*).

Используйте этот файл как **короткий указатель** при планировании; детали и критерии готовности — только в [roadmap-checklist.md](roadmap-checklist.md).
