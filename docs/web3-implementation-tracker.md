# Web3 / BET — трекер внедрения (инженерный)

Канон дизайна: [crypto-economy.md](crypto-economy.md) §2–3, [bet-whitepaper-v0.md](bet-whitepaper-v0.md) (фазы **A–H**). Этот файл — **очередность работ в репозитории** и якоря для статуса; смарт-контракты и индексатор появятся в отдельных каталогах по мере реализации.

| Фаза | Содержание | Соответствие crypto-economy | Статус в репо |
|------|------------|----------------------------|---------------|
| **A** | Продуктовый/юридический sign-off параметров v0; выбор тестовой сети | Whitepaper v0 → **v1** перед публичными средствами | Вне репозитория |
| **B** | BET **ERC-20** в тестнете (эмиссия, mint от игрового ядра, публичный burn) | **§2.1** (минимальный первый шаг) | Добавлены bootstrap-артефакты: `docs/web3-phase-a-signoff-checklist.md`, `backend/docs/bet-erc20-reference.sol` |
| **C** | Индексатор событий → **PostgreSQL** / **Redis**; RPC/BFF для gateway | **§2.3** | Стартовый ingest-сервис `backend/cmd/web3-indexer`, миграция `20260501120000_web3_indexer_foundation.sql` |
| **D** | Газлесс-релей | **§2.3** | Сервис **TBD** |
| **E** | Роли, pause, anti-whale, нагрузочные прогоны | **§2.4** | После B–D |
| **F** | API/Unity: embedded wallet, покупки, аукцион, зоны | **§2.2** | Поверх синхронизированной БД из **C** |
| **G** | Публичные адреса контрактов, дашборд казны | **§2.5** | Mainnet / публичный запуск |
| **H** | Внешний аудит контрактов | **§2.4** / **§2.5** | Перед mainnet/листингом |

## Рекомендуемая зависимость (как в §2)

```text
A → выбор сети → B → C → D → E → F → G → H
```

Параллельно **B–D** допустима подготовка **схем БД** и контрактов миграций goose под ончейн-состояние (без приёма средств), по описанию в [bet-whitepaper-v0.md §2](bet-whitepaper-v0.md).

## Следующий инженерный коммит (после фазы A)

1. Вынести референс-контракт в отдельный каталог контрактов (Foundry/Hardhat) и задеплоить **BET** в выбранном тестнете.
2. Зафиксировать адреса и ABI в конфиге; подключить источник событий к `POST /v1/indexer/ingest`.
